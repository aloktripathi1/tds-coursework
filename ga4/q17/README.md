# Q17: The Hallucination Matrix

## Task

Given 1,000 Python scripts generated by a rogue LLM, identify the **single valid script** among them. 999 scripts contain subtle hallucinations — calls to methods that sound plausible but don't actually exist in the Python standard library or common third-party libraries (`pandas`, `requests`, etc.).

**Example hallucination:**
```python
# Fake
df.drop_nulls()
# Real
df.dropna()
```

---

## Requirements

* Scan all 1,000 scripts in the `scripts/` folder
* Detect hallucinated method calls (e.g., `json.parse`, `os.path.build_path`, `df.group_by`)
* Identify the one script that is 100% syntactically and logically valid
* Output: the filename of the correct script (e.g., `script_042.py`)

---

## Approach

### AST-Based Hallucination Detection

The solution uses Python's `ast` module to statically analyze each script and flag hallucinated method names:

1. **Parse each file** into an AST — any file with a `SyntaxError` is immediately flagged.
2. **Walk the AST** looking for `ast.Attribute` nodes whose `.attr` matches a curated list of ~70 known hallucinated method names across common libraries:
   - **requests/HTTP:** `fetch`, `download`, `get_json`, `get_content`, `retrieve`, etc.
   - **os/os.path:** `build_path`, `join_path`, `create_directory`, `file_exists`, etc.
   - **json:** `parse`, `stringify`, `to_string`, `dump_str`, `ParseError`, etc.
   - **datetime:** `from_string`, `set_month`, `set_date`, etc.
   - **pandas:** `drop_nulls`, `group_by`, `filter_rows`, `load_csv`, `rename_columns`, etc.
   - **file/io/dict:** `read_all`, `read_to_string`, `get_or_default`, etc.
3. **Validate keyword arguments** for known functions (e.g., `timedelta` with invalid kwargs).
4. **Files with zero detections** are reported as clean — exactly one file passes all checks.
